SOLID Principles

1. Single Responsability A class or object has only one reason to change.

2. Open/Closed The classes are open to extension and close to modification.

3. Liskov Substitution Should be possible use a parent in the place of the child class without broke the project.

4. Interface Segregation Keep interfaces small. Don't make a classe implemente some method with params that it is not necessary

5. Dependency Inversion Classes should depend of abstraction, not concrete classes
